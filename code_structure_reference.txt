RESCUE DRONE v3 - CODE STRUCTURE REFERENCE
==========================================

This document outlines the architecture and purpose of every file within the project, which is structured to completely isolate the AI, the UI, and the Hardware interfaces.

1. Root Directory (The Core)
----------------------------
* main.py: The primary executable stringing it all together. It parses your command-line arguments, loads up your settings, creates the detection pipeline, fires up the display window, and runs the infinite loop that drives the app.
* config.py: Your centralized "brain settings" file. It stores everything: pin connections, UART ports, YOLO confidence thresholds, sensor fusion weights, and display constants. Keeping this separate makes tweaking thresholds easy.
* rescue_drone.service: A systemd file strictly meant for Linux OS. Once installed, it allows the Pi to automatically execute main.py natively in the background the exact second the battery is plugged in.
* requirements.txt: A list of all external Python libraries (like OpenCV, Numpy, etc.) required to make your environment run.
* README.md: Your main documentation markdown file outlining how exactly the repo is structured for viewers.

2. pipeline/ Directory (The Traffic Controller)
-----------------------------------------------
* detection_pipeline.py: This is the real “engine room.” On every single new frame, this script queries all connected hardware sensors, feeds those raw frames to your AI models, executes auto-snapshot logic, and finally bundles the finished metrics into a neat packaged object (FrameData) for the display to render.

3. display/ Directory (The Visual Output)
-----------------------------------------
* rescue_display.py: Controls what actually goes to your screen (or your analog VTX!). It builds the OpenCV Fullscreen canvas, composites the Picture-in-Picture thermal overlays, handles togglable main views (RGB vs Thermal), and intercepts your keyboard shortcuts (like generating a snapshot when you press S).
* osd.py: Dedicated entirely to rendering the On-Screen Display. It draws the literal text telemetry lines, the bounding boxes clamping onto targets, the crosshairs, and dynamically generates the radar scope visualization based on LD2450 target plotting.

4. ml/ Directory (Machine Learning & Logic)
-------------------------------------------
* models.py: Contains several massive interconnected classes. 1) The YOLO detector (for RGB). 2) An Anomaly Detector (to search for unexplained thermal blooms). 3) A HumanTracker (using the SORT algorithm to give spotted people unique IDs). 4) SensorFusion (which decides if an RGB body + Thermal Blob + Radar Ping equals a confirmed survivor).
* thermal_isolation.py: Specific image-processing math that manipulates the raw thermal grid. Depending on the mode selected, it will try to clip background noise below body-temperature limits and isolate human silhouettes or glowing contour forms visually.
* detection.py: A very lightweight file that likely just stores the strict definition/variables of what a Detection actually is across the code (bounding box coordinates, confidence score, source sensor, temperature, etc.).

5. sensors/ Directory (The Hardware Interfaces)
-----------------------------------------------
* rgb_camera.py: Communicates exclusively with the Pi Camera Module 3 over the libcamera backend, grabbing the primary visible-light video frames.
* thermal_camera.py: Specifically handles the I2C protocol bridging your MLX90640. It translates 768 individual thermal pixel pulses into color-mapped numpy arrays.
* ld2450_radar.py: Standard UART serial decoder strictly meant to talk back and forth with the HLK-LD2450 mmWave presence module and read ranges/movement metrics.
* flight_controller.py: A MAVLink telemetry decoder bridging your RPi to the F722. It passively monitors OSD stats like drone altitude, voltage, and pitch to be forwarded down to the display.

6. The Helper Folders
---------------------
* models/: The storage locker for your compiled .onnx YOLO weights. 
* output/: The designated save location where your app dumps manual snapshots, automated SORT tracker 5-second interval snapshots, and video recordings.
* config/: Simply a failsafe directory generated by config.py.
